# Trading Operating System

## Vision
Trading Operating System (TOS) is a disciplined risk-management framework for MetaTrader 5. The project is designed to protect capital, enforce trading discipline, collect structured trading data, and later support AI-assisted trading workflows without sacrificing control.

## Goals
- Protect capital through explicit risk controls.
- Enforce disciplined trading behavior and pre-trade safeguards.
- Collect trading data for future analysis and workflow automation.
- Provide a modular, clean-architecture foundation for future extensions.

## Phase-Based Roadmap
### Phase 1 - Foundation
- Create the project structure and documentation skeleton.
- Define the modular MQL5 include architecture.
- Establish placeholder modules and configuration-presets structure.

### Phase 2 - Risk Controls
- Implement core risk-management rules and guardrails.
- Add monitoring and operational safeguards.
- Introduce structured logging and reporting placeholders.

### Phase 3 - Data Collection
- Capture execution and risk data for review.
- Prepare the system for future analytics and model integration.

### Phase 4 - AI-Assisted Workflows
- Add optional decision-support workflows.
- Keep human oversight and capital protection central.

## Current Phase Status
Current status: Phase 1 in progress.

This repository currently contains the structural foundation, documentation skeleton, and placeholder modules only. No trading strategy logic, indicators, or automated execution logic has been implemented.
